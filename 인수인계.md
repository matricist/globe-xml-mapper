# Globe XML Mapper 인수인계

## 뭐 하는 프로그램

국조53/55/56 서식(엑셀)을 OECD GloBE 정보신고서 XML(GIR)로 바꿔주는 WinForms 앱.
사용자가 통합 템플릿 한 파일(`Resources/main_template_newest.xlsx`)에 데이터 채워 넣고 변환 누르면 `GLOBE_OECD.xml`이 떨어진다.

## 코드 구조

소스는 사실상 `Services/` 안에 다 있다. 매퍼 클래스 하나가 시트 하나를 담당하는 식으로 1:1 분리:

- `Mapping_1.1~1.2.cs` → MNE 그룹 정보 시트
- `Mapping_1.3.1.cs` → 최종모기업
- `Mapping_1.3.2.1.cs` → 그룹구조(CE)
- `Mapping_1.3.2.2.cs` → 제외기업
- `Mapping_1.3.3.cs` → 그룹구조 변동
- `Mapping_1.4.cs` → 요약
- `Mapping_2.cs` → 적용면제
- `Mapping_JurCal.cs` → 국가별 계산 (3.1~3.3 전부)
- `Mapping_EntityCe.cs` → 구성기업 계산 (3.2.4 + 3.4)
- `Mapping_Utpr.cs` → UTPR 배분

`MappingOrchestrator.cs`가 위 매퍼들을 순서대로 돌리고 마지막에 후처리(MessageSpec/DocSpec 채움 + XSD 필수필드 가드)까지 한다.

매퍼 안에서 자주 쓰는 공통 유틸은 `MappingBase.cs`에 모여 있다 — `ParseTin`, `ParsePercentage`, `TryParseDate`, `NullIfEmpty` 등.

## AI한테 처음 던질 프롬프트

이게 제일 중요한데, 후임자가 코드 다 안 보고 AI에 맡길 거면 처음에 이거 먹이면 거의 끝난다:

```
CLAUDE.md, Resources/Globe요약.md, Services/MappingBase.cs,
Services/MappingOrchestrator.cs를 읽고 이 프로젝트의 변환 흐름을
요약해줘
```

- `CLAUDE.md` — 시트별 행/열 매핑, 통합셀 포맷, 비율 파싱 규칙, 빈 XML 회피 규칙 등 도메인 지식 다 정리되어 있음. 우리 프로젝트의 한장 요약 같은 거.
- `Globe요약.md` — XSD 자동생성된 `Globe.cs`(13000줄)의 압축본. AI한테 직접 `Globe.cs` 던지면 토큰만 잡아먹으니 이걸 보게 해야 함.
- `MappingBase.cs` + `MappingOrchestrator.cs` — 진입점 + 공통 패턴.

이거 네 개면 어지간한 작업 다 가능.

## 자주 사고나는 지점

**1. 사용자가 `%` 단위로 입력함**
"5%"나 "15" 같은 거. XSD는 0~1 decimal만 받기 때문에 `ParsePercentage`가 엄격 모드로 거부한다. 사용자한테 "0.05로 입력하세요"라고 안내해야 함. 에러 메시지에도 그렇게 나오게 해뒀다.

**2. xlsx 수식이 평가가 안 된 채로 올라옴**
사용자가 Excel에서 한번도 저장 안 하고 업로드하면 수식 셀의 캐시값이 비어 있다. `MappingOrchestrator`가 `RecalculateAllFormulas()` 부르긴 하는데 ClosedXML이 처리 못하는 함수(`_xlfn.CONCAT` 같은)가 있으면 일부 누락 가능. 그럴 땐 그냥 "Excel에서 한번 열고 저장 후 재시도" 안내.

**3. 템플릿 행 위치가 바뀌면 매퍼가 깨짐**
그래서 `Mapping_*.cs` 어디에서도 절대 행번호 쓰지 않는다. 무조건 `FindRow("헤더텍스트")` + 상대 오프셋. 새 매퍼/시트 추가할 때 같은 원칙 지킬 것. 한번이라도 절대 행번호 박으면 다음 서식 개정 때 무조건 깨진다.

**4. XSD [R] 필드 누락**
사용자가 셀 안 채우면 그 필드 통째로 emit 안 되는 경우가 있는데, 부모는 만들어졌으니 XSD가 `<sequence>` 위반이라고 잡아낸다. `MappingOrchestrator.EnsureXsdRequiredFields()`가 0 기본값으로 채워주는 후처리를 하니 새 [R] 필드 추가될 때마다 거기에 등록해야 함.

**5. Excel "백분율" 셀 서식**
셀이 백분율 서식이면 `GetString()`이 "5%" 같은 형식 문자열을 돌려준다. 그러면 ParsePercentage가 거부함. 템플릿 셀 서식은 "숫자"로 통일하는 게 맞다. (현재 템플릿은 정리되어 있음.)

## 검증

`Services/ValidationUtil.cs` — OECD가 정의한 70xxx 비즈니스 룰 검증. 변환 시 자동 호출. 전부 구현된 건 아니고 자주 보는 것만. 미구현 코드 목록은 CLAUDE.md에 명시되어 있다.

`MappingOrchestrator.EnsureXsdRequiredFields()` — 매핑 후처리. XSD [R] 필드가 사용자 입력 누락 등으로 빠지지 않도록 0 기본값으로 보장.

매핑 결과는 GUI에서 변환 시 `<출력파일>.errors.txt`로 떨어진다.

## 빌드 / 배포

- 빌드: `dotnet build GlobeMapper.csproj`
- 배포: `publish/` 폴더 (csproj에 publish 프로파일 들어 있음)
- 인증서/만료일은 `Resources/activation_config.json`에 박혀 있음. 이거 갱신 안 하면 사용자 쪽에서 만료 메시지 뜸.

## 마지막으로

매핑 규칙이 워낙 많아서 처음 열어보면 좀 압도당할 수 있는데, 시트 하나당 매퍼 하나, 행 위치는 헤더 텍스트로 동적 탐색, XSD [R]은 후처리에서 보장 — 이 세 원칙만 잡고 보면 생각보다 단순하다. 의문 생기면 CLAUDE.md → Globe요약.md → 해당 매퍼 순서로 추적하면 거의 다 풀린다.

XSD 통과까지는 안정적이고, 그 다음 단계인 OECD 의미 검증(70xxx)은 ValidationUtil 커버리지가 100%가 아니라 추가 작업 여지 있음. 사용자가 신고하면서 잡힌 코드들은 모아뒀다가 우선순위대로 추가하면 됨.
